PREMESSA CHE SPIEGA DIVERSI ELEMENTI NON RICONOSCIUTI NEL PARSING:
LA STRUTTURA DEGLI SCRIPT è UNA STRUTTURA A BLOCCHI.
Lo script può quindi avere dei blocchi al cui interno ci sono altri blocchi o comandi singoli.
TUTTO QUANTO IN UNO SCRIPT è INSERITO TRA I DUE BLOCCHI [SCRIPT <nomescript>]...[END_OF_SCRIPTS]
Quali sono gli altri blocchi?
1) tutti i tipi di IF Standard e Predefiniti: IF, IF_DEBUG, IF_FROM_CAMPAIGN, IF_HAS_CREDITS, IF_IS, IF_MAX, IF_MIN, IF_MISSION_WON, IF_ORDER, IF_PROB, IF_TUTORIAL_SEEN, IFMISSIONRESULTIS, IFMISSIONRESULTMIN, IFNOT) che rappresentano un blocco fino ad [END_OF_IF]
   Per tutti gli IF inoltre è data la possibilità di attivare il ramo [ELSE]. Se ELSE è attivo qualsiasi elemento IF conterrà due blocchi distinti e paralleli di elementi contenuti: [IF] contenuto blocco 1 [ELSE] contenuto blocco 2 [END_OF_IF]
2) i MENU che sono definiti con [MENU]... [END_OF_MENU] E AL LORO INTERNO POSSONO CONTENERE SOLO ELEMENTI DI TIPO OPT e devono essere preceduti a livello di flusso da un ASK
3) tutti i tipi di OPT Standard o OPT con verifica su Semaforo [OPT_IF] e [OPT_IFNOT] sono delimitati nel loro blocco da [END_OF_OPT]

- Proprietà: nome script, parametri, contenuto

### 2. MISSION BLOCKS  
- **MISSION** <nome> ... **FINISH_MISSION** ... **END_OF_MISSION**: Container missione
- Sottoblocchi specializzati:
  - **INIT_BUILD** ... **START_BUILDING** ... **END_BUILDING**
  - **INIT_FLIGHT** ... **START_FLIGHT** ... **EVALUATE_FLIGHT** ... **END_FLIGHT**

### 3. CONDITIONAL BLOCKS (Famiglia IF)
- **IF** <condizione> ... [**ELSE**] ... **END_OF_IF**
- Varianti specializzate:
  - **IF_DEBUG**, **IF_FROM_CAMPAIGN**, **IF_HAS_CREDITS** <amount>
  - **IF_IS** <var> <value>, **IF_MAX** <var> <value>, **IF_MIN** <var> <value>
  - **IF_MISSION_WON**, **IF_ORDER** <positions>, **IF_PROB** <percentage>
  - **IF_TUTORIAL_SEEN**, **IFMISSIONRESULTIS** <result>, **IFMISSIONRESULTMIN** <min>
  - **IFNOT** <semaforo>
- Tutti supportano ramo **ELSE** opzionale

### 4. MENU BLOCKS
- **MENU** ... **END_OF_MENU**: Container menu interattivo
- Preceduto sempre da **ASK** <domanda>
- Contiene solo elementi **OPT** family

### 5. OPTION BLOCKS (Famiglia OPT - solo dentro MENU)
- **OPT** <testo> ... **END_OF_OPT**: Opzione base
- **OPT_IF** <semaforo> <testo> ... **END_OF_OPT**: Opzione condizionale (se ON)
- **OPT_IFNOT** <semaforo> <testo> ... **END_OF_OPT**: Opzione condizionale (se OFF)

## 🎨 INTERFACCIA VISUAL EDITOR - DESIGN PATTERNS

### 1. PALETTE STRUMENTI
Organizzata per categorie funzionali:
- **📦 Containers**: Script, Mission, IF blocks, Menu, Options  
- **💬 Dialog**: Say, Ask, ShowChar, HideChar, ChangeChar
- **🔀 Flow**: Label, Go, SubScript, Return, Delay
- **📊 Variables**: Set, Reset, Set_To, Add
- **🗺️ Map**: ShowNode, ShowPath, MovePlayer, CenterMap
- **🎮 UI**: ShowButton, SetFocus, Announce
- **🚀 Mission**: AddOpponent, SetMissionAs*, AddCredits
- **⚙️ System**: SaveState, QuitCampaign, specialized setters

### 2. VALIDAZIONE CONTESTUALE
- **Dialog commands**: Validare se finestra dialogo è aperta
- **Menu options**: OPT family solo dentro MENU blocks
- **ASK prerequisite**: MENU deve essere preceduto da ASK
- **Script nesting**: Limitare profondità annidamento
- **Variable scope**: Tracking semafori/variabili utilizzate

### 3. VISUALIZZAZIONE FLUSSO
- **Container blocks**: Rendering espandibile con header/footer
- **Conditional branches**: Visualizzazione a due rami (IF/ELSE)
- **Menu trees**: Struttura ad albero con opzioni
- **Flow arrows**: Connessioni GO->LABEL, SUB_SCRIPT->Script
